**Quick Summary:**
- **No return:** Only `forEach()` (always `undefined`).
- **Return new array:** `map`, `filter`, `flat`, `slice`, `concat`, `toSorted`, `toReversed`, etc.
- **Return single value:** `reduce`, `find`, `findIndex`, `indexOf`, etc.
- **Return boolean:** `some`, `every`, `includes`.
- **Return string:** `join`, `toString`.
- **Return iterator:** `keys`, `values`, `entries`.
- **Return mutated array:** `sort`, `reverse`, `splice`, `fill`, `copyWithin`.

**Rule of Thumb:**
- Use **arrow functions (callbacks)** when a method needs logic per element (map, filter, reduce, etc.).
- No arrow function is needed when the method only manipulates/returns values without conditions (`concat`, `slice`, `reverse`, etc.).
