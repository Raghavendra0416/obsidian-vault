### What is `opacity`?
`opacity` controls **how see-through an element is**.

- Values go from **`0` (fully transparent)** → **`1` (fully opaque)**.
- The **default** is **`1`**.
- It’s **animatable** (you can fade things in/out). 

```css
.box { opacity: 0.5; }  /* 50% see-through */
```

**Important:** Using `opacity` on a parent makes **everything inside it** (text, images, buttons) transparent too. If you only want a transparent **background** but solid text, use an RGBA color instead of `opacity`. 

```css
/* Bad if you want readable text: children also fade */
.banner { opacity: 0.6; }

/* Good: only the background is translucent, text stays solid */
.banner { background: rgba(0, 0, 0, 0.6); color: white; }
```

Calls this out directly and shows examples of both patterns. 
### Why learn it?
- **Visual focus:** dim things you want de-emphasized (e.g., background behind a modal).
- **States:** make disabled buttons look inactive.
- **Effects:** quick **hover fades** and smooth **entrance/exit** animations. 

### Where is it used?
- **Hover effects** on images/cards
- **Overlays** (darken the page behind a dialog)
- **Steppers/loaders** (fade elements in sequence)  
    All major browsers support `opacity` (it’s been around for ages). 

### Tiny, easy examples

#### 1) Image hover fade

```css
img.thumb { transition: opacity .2s; }
img.thumb:hover { opacity: 0.7; }
```

Hovering makes the image subtly fade. (The page’s demo shows changing `opacity` values.)

#### 2) Disabled button look

```css
button[disabled] { opacity: 0.5; cursor: not-allowed; }
```

Half-transparent + a “not allowed” cursor sells the disabled state.

#### 3) Readable banner with translucent background

```css
.banner {
  background: rgba(0, 0, 0, 0.5);  /* only bg is translucent */
  color: #fff;
  padding: 1rem;
}
```

This uses **RGBA** so the text stays crisp (exactly the workaround W3Schools suggests). 

### Good to know
- **Syntax:** `opacity: number;` and you can set it from JS as `element.style.opacity = "0.5"`. 
- **Not inherited** (the property itself doesn’t pass down), but the **visual effect** applies to the whole element—including children.
