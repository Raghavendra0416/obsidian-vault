**“In CSS, the `display` value controls how an element participates in layout. The three common ones are `block`, `inline`, and `inline-block`.**

- **Block elements (`display: block`)**
    - **Line breaks:** Start on a new line and take up the full available width by default.
    - **Sizing:** You can set **width/height**. Default width is stretch (“auto” = fill container).
    - **Margins/Padding:** All sides respected; **vertical margins can collapse** with adjacent blocks (e.g., parent/child, siblings).
    - **Context:** Establish a **block formatting context** for their children (standard flow).
    - **Examples:** `div`, `p`, `h1–h6`, `ul/ol`, `li`, `section`, `article`.
    - **Use when:** You want stacked sections/rows that span the container.
        
- **Inline elements (`display: inline`)**
    
    - **Line breaks:** Flow **within a line** like text; **no line break** before/after by default.
    - **Sizing:** **Can’t set width/height** (ignored); box is sized by its content.
    - **Margins/Padding:** **Horizontal** margins/padding affect layout; **vertical** ones don’t push lines apart (often ignored in layout).
    - **Alignment:** Align to the **baseline** of the line box; controlled by `line-height`/`vertical-align`.
    - **Examples:** `span`, `a`, `em`, `strong`, `code`.
    - **Special case:** **Replaced inline elements** (e.g., `img`, `input`, `video`) are inline but **do accept width/height**.
    - **Use when:** You’re styling text and small, in-flow fragments.
        
- **Inline-block elements (`display: inline-block`)**
    
    - **Line breaks:** Sit **in line** (like inline) so multiple can appear on one line.
    - **Sizing:** **Can set width/height** (like block). Width is **shrink-to-fit** by default.
    - **Margins/Padding:** All sides respected; **no margin collapsing** with children.
    - **Alignment:** Participate in the inline line box; **baseline** aligned by default, tweakable with `vertical-align`.
    - **Context:** Behaves inline on the outside; inside it forms a **block formatting context** for its content.
    - **Gotcha:** **Whitespace between inline-block elements** in HTML creates visible gaps (because they’re inline).
    - **Use when:** You want items side-by-side like text, but with block-like sizing (badges, buttons, cards).
        

**Common comparisons**

- **Line break behavior:** block = breaks line; inline/inline-block = stay on the same line if space allows.
- **Set width/height:** block ✅, inline ❌ (except replaced), inline-block ✅.
- **Vertical margins:** block ✅ (and may **collapse**), inline ❌ (don’t affect line flow), inline-block ✅ (no collapsing).
- **Alignment controls:** inline & inline-block respond to `vertical-align`; blocks don’t participate in line-box alignment.

**How to switch:** `display: block | inline | inline-block;` depending on layout needs (e.g., turn a `span` into a block or make a `div` inline-block for pill-like components).

**Quick rule of thumb:**

> Use **block** for stacked sections, **inline** for text-level elements, and **inline-block** when you need text-like placement with controllable dimensions.”

|Aspect|**Block (`display: block`)**|**Inline (`display: inline`)**|**Inline-block (`display: inline-block`)**|
|---|---|---|---|
|**Line breaking**|Starts on a **new line**; occupies its own row|Flows **within a line** like text|Flows **within a line** (can sit side-by-side)|
|**Default width**|**Fills** available width|**Content-sized**|**Shrink-to-fit** content (until constrained)|
|**Set `width`/`height`**|✅ Supported|❌ Ignored (except **replaced** elements like `img`, `input`)|✅ Supported|
|**Margins & padding**|All sides apply; **horizontal & vertical** affect layout|**Horizontal** margins/padding apply; **vertical margins are not applied** (top/bottom padding/border can enlarge the line box)|All sides apply; **no margin collapsing**|
|**Margin collapsing**|**Can collapse vertically** with adjacent blocks|N/A|**Does not collapse**|
|**Alignment**|Not in a line box; uses normal block flow|Aligns to the **baseline** of the line; controllable via `vertical-align`|Aligns in the line box (baseline by default); **`vertical-align`** works|
|**Formatting context**|Acts as a block container for children|Inline formatting context (text runs)|Inline **outside**, block **inside** (forms its own box)|
|**Examples**|`div`, `p`, `h1–h6`, `ul`, `li`, `section`, `article`|`span`, `a`, `em`, `strong`, `code` (plus **replaced inline**: `img`, `video`)|Commonly applied via CSS to `button`-like or badge-like components|
|**Typical use**|Stacked sections/rows, layout regions|Text-level elements, small in-flow fragments|Side-by-side items with controllable dimensions (buttons, pills, cards)|

**Rule of thumb:** use **block** for stacked layout sections, **inline** for text-level content, and **inline-block** when you need text-like placement **with** controllable width/height.
