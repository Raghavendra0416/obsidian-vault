#### Session 1:
- 2 Types of data-
	 - Primitive (Numbers, Strings, Booleans)
	 - Non-Primitive
- Variables are not values. Variables are just labels pointing to values store in the memory.
- Var- used to declare variables. Re-assignment is allowed. Re-declaration is allowed. (Not used the most).
- Let- used to declare variables. Re-assignment is allowed. Re-declaration is not allowed.
- Const- Can only be declared once and cannot be changed afterwards. Re-assignment is not allowed. Re-declaration is not allowed.
- Values are stored in the memory, and variables are the labels pointing to the memory address where the value is stored.
- When we assign a value to a variable either directly (e.g. b = 20;) or from one variable to another. (e.g. b = a;), a new copy of that value is created and stored in the memory and the variable points to it.

-  We have " == " and " === " in JS 
		== -> loose comparison, before comparing covert both the values to same types and will compare them. So if int(5) == ('5') String gives true.
		=== -> Strict comparison, Will compare datatype and values. So, is int(5) === ('5') String will return false. 
		Example: (0-False; 1- True; According to JS)
			0 == false --> true
			0 === false --> false 
- What is Argument & parameter.

	**Parameter-** the values we pass when we are defining a function is known as **"Parameters"**.
