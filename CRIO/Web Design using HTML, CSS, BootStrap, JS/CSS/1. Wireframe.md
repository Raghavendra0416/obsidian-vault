### Whatâ€™s a wireframe?
Think of a wireframe like a **blueprint for an app or website screen**.  
Itâ€™s a quick, simple drawing that shows **where things go** (header, menu, buttons, forms) and **how you move** from one screen to another. No colors, no fancy fontsâ€”just boxes, labels, and arrows.

### Why designers use wireframes

- **Fast & cheap**: you can sketch 5 ideas in minutes.
- **Focus on the idea**: people react to the _flow_ and the _layout_, not the colors.
- **Easy to change**: erasing a box is way easier than redoing polished screens.

### Low-fi vs High-fi (plain English)

- **Low-fidelity (low-fi)** = rough sketch. Boxes, scribbles, placeholder text.
- **High-fidelity (high-fi)** = polished mockup. Real fonts, colors, spacing, near-final look.  
    Start low-fi to nail the concept, then go high-fi when the flow feels right.

- **Circles/rounded boxes** for buttons
- **Arrows** to show navigation between screens
- Tiny **notes** if something needs explaining
